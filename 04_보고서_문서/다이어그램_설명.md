## 1️⃣ System Architecture (시스템 아키텍처)

**"6-Layer 구조의 End-to-End 데이터 파이프라인 설계"**

### 📊 상세 정보

#### **Layer 1: 데이터 소스 (Data Source)**
- **서울 열린데이터광장 (CCTV)**: 24개 자치구 CCTV 설치 현황 데이터
  - 방범용, 교통단속, 어린이보호구역 등 목적별 분류
- **공공데이터포털 (범죄통계)**: 5대 범죄 발생 현황
  - 살인, 강도, 강간강제추행, 절도, 폭력 건수
- **통계청 KOSIS (인구 통계)**: 자치구별 등록인구 데이터
  - 총인구, 세대수, 고령자수 등

#### **Layer 2: 데이터 수집 (Data Collection)**
- **API 호출 모듈**: `fetch_seoul_data.py`
  - REST API 기반 자동 데이터 수집
  - `.env` 파일로 API 키 관리 (보안)
- **원본 데이터 저장**: `data/raw/` 폴더
  - CSV 형식으로 저장 (UTF-8 인코딩)

#### **Layer 3: 데이터 처리 (Data Processing)**
- **데이터 정제**: `process_real_data.py`
  - 결측값 처리: '-' → 0 변환
  - 자치구명 표준화: "종로구청" → "종로구"
  - 데이터 타입 변환: 문자열 → 숫자
- **데이터 통합**: Pandas `merge()` 연산
  - Inner Join 방식으로 3개 데이터셋 병합
  - 최종 24개 자치구 매칭 완료
- **정제 데이터 저장**: `data/processed/` 폴더

#### **Layer 4: 분석 (Analysis)**
프로세스가 4개 병렬로 진행됨:

1. **탐색적 데이터 분석 (EDA)**
   - 기술 통계: 평균, 중앙값, 표준편차, 사분위수
   - 분포 확인: 히스토그램, 박스플롯

2. **그래프 시각화**
   - Matplotlib: 산점도, 막대차트
   - Seaborn: 상관계수 히트맵
   - 정적 이미지 저장 (PNG, 300 DPI)

3. **회귀 분석**
   - 다중 선형 회귀: Y = β₀ + β₁X₁ + β₂X₂ + ε
   - 가정 검토: VIF, Q-Q Plot, 등분산성
   - R² = 0.627, p < 0.0001

4. **지역 분류**
   - 4사분면 분류 알고리즘 (자세한 내용은 다이어그램 2 참고)
   - 정책 우선순위 도출

#### **Layer 5: 시각화 (Visualization)**
- **분석 리포트**: `results/reports/`
  - 텍스트 기반 분석 요약 (TXT)
  - 통계량, 상관계수, 상위 자치구 정보

- **인터랙티브 대시보드**: `dashboard_real.py`
  - Streamlit 프레임워크 사용
  - 5개 탭 구성: 개요, CCTV 분석, 범죄 분석, 상관관계, 데이터 테이블
  - Plotly 인터랙티브 차트

- **결과 파일**: `results/figures/`
  - 산점도, 히트맵, 막대차트 등

#### **Layer 6: 배포 (Deployment)**
- **GitHub Repository**: 버전 관리 및 코드 공개
  - README.md: 프로젝트 문서화
  - 다이어그램, 분석 결과 포함

- **Streamlit Cloud**: 대시보드 배포
  - 공개 URL 생성 (`https://your-app.streamlit.app`)
  - 무료 호스팅, CI/CD 자동화

## 2️⃣ 4사분면 분류 알고리즘 (Quadrant Classification)

**"중앙값 기반 자치구 분류 및 정책 우선순위 도출"**

### 📊 상세 정보

#### **Step 1: 시작 (Start)**
프로세스 시작점

#### **Step 2: 통합 데이터 입력**
- **입력 데이터**: `integrated_data_with_analysis.csv`
- **주요 컬럼**:
  - `자치구`: 지역명 (24개)
  - `방범CCTV_per_1000`: 인구 천명당 방범 CCTV 대수
  - `CCTV효과범죄_per_1000`: 인구 천명당 CCTV효과범죄 건수 (절도+강도)
  - `총인구`, `CCTV_총계`, `총범죄_발생` 등

#### **Step 3: 중앙값 계산 (Median Calculation)**

**코드 구현**:
```python
cctv_median = df['방범CCTV_per_1000'].median()
crime_median = df['CCTV효과범죄_per_1000'].median()
```

**중앙값을 사용하는 이유**:
- **평균의 문제점**: 강남구 같은 극단값(Outlier)에 영향 받음
- **중앙값의 장점**: 정확히 50%씩 분할, Robust(강건함)

**실제 값 예시**:
```
CCTV 중앙값 = 11.5대/천명
범죄 중앙값 = 3.6건/천명
```

#### **Step 4: 등급 분류 (Grade Classification)**

각 자치구를 중앙값과 비교하여 등급 부여:

```python
def classify_grade(row):
    cctv_grade = 'High' if row['방범CCTV_per_1000'] >= cctv_median else 'Low'
    crime_grade = 'High' if row['CCTV효과범죄_per_1000'] >= crime_median else 'Low'
    return cctv_grade, crime_grade
```

**등급 예시**:
- 강남구: CCTV High, 범죄 High
- 도봉구: CCTV Low, 범죄 Low

#### **Step 5: 4사분면 할당 (Quadrant Assignment)**

등급 조합에 따라 4개 사분면으로 분류:

```python
def classify_quadrant(row):
    cctv = row['방범CCTV_per_1000']
    crime = row['CCTV효과범죄_per_1000']

    if cctv >= cctv_median and crime >= crime_median:
        return 'Q1: 고CCTV/고범죄'
    elif cctv < cctv_median and crime >= crime_median:
        return 'Q2: 저CCTV/고범죄 (우선순위)'
    elif cctv < cctv_median and crime < crime_median:
        return 'Q3: 저CCTV/저범죄'
    else:
        return 'Q4: 고CCTV/저범죄 (효과적)'
```

---

### 🔍 4사분면 상세 설명

#### **1사분면 (Q1): 고CCTV/고범죄**
- **특징**: CCTV 많지만 범죄도 많음
- **해석**: CCTV만으로 부족, 추가 대책 필요
- **예시 자치구**: 강남구, 송파구, 관악구
- **정책 제안**:
  - 경찰 순찰 강화
  - LED 가로등 교체 (조명 개선)
  - 주민 안심귀가 서비스
  - CCTV 화질 업그레이드

#### **2사분면 (Q2): 저CCTV/고범죄 ⚠️ 최우선 순위**
- **특징**: CCTV 부족하고 범죄 많음
- **해석**: CCTV 추가 설치 최우선 지역
- **예시 자치구**: 영등포구, 동대문구, 광진구
- **정책 제안**:
  - 방범 CCTV 집중 투자
  - 범죄 다발 지점 우선 설치
  - 분기별 효과 모니터링

#### **3사분면 (Q3): 저CCTV/저범죄**
- **특징**: CCTV 적지만 범죄도 적음
- **해석**: 현 상태 유지, 불필요한 투자 지양
- **예시 자치구**: 도봉구, 노원구, 강북구
- **정책 제안**:
  - 현재 수준 유지
  - 범죄 증가 추세 모니터링
  - 예산 절약

#### **4사분면 (Q4): 고CCTV/저범죄**
- **특징**: CCTV 많고 범죄 적음
- **해석**: CCTV가 효과적으로 작동
- **예시 자치구**: 서초구, 용산구, 종로구
- **정책 제안**:
  - 모범 사례로 분석
  - CCTV 배치 전략 공유
  - 다른 자치구에 적용

# 분면 컬럼 추가

Q3 (저CCTV/저범죄): 8개 자치구 (33%)
Q4 (고CCTV/저범죄): 6개 자치구 (25%)
