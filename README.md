## 문제 정의 & 해결 방향

### 문제 정의

일반적으로 CCTV 수가 많으면 범죄가 줄어들 것이라 기대하지만, 실제로는 **범죄가 많아서 CCTV가 뒤늦게 설치되는 역인과관계가 존재**할 수 있습니다.

### 해결 방향

서울시 공공데이터를 통해 CCTV 밀도와 범죄 발생 간의 관계를 정량적으로 검증하고, 정책 해석의 오류를 줄이기 위한 근거를 마련하는 것을 목표로 했습니다.

## 분석 방법

### 상관분석 및 회귀분석

- 24개 자치구 × 36개 변수로 구성된 데이터 처리
- CCTV 대수와 범죄 건수 간 상관계수 산출

### 4사분면 기반 위험도 분류

- 인구당 CCTV 수 × 인구당 범죄율 기준으로 자치구를 4개 그룹 자동 분류
- Z-score 기반 위험도 점수화 모델 구축

---

## 핵심 결과

### 1. 역인과관계 확인

- CCTV 설치 상위 5개 자치구 = 범죄 발생 상위 5개 자치구
- CCTV가 "예방용"이 아니라 **사건 이후에 따라가는 형태**로 배치

### 2. CCTV 우선 설치 지역 도출

- **Q2 (저CCTV/고범죄)**: 4개 구, 평균 6.60대(1,000명당) — **최우선 설치 대상**
- 중앙값(8.34대) 대비 **−20.9% 격차**

### 3. 자치구 간 변동성

- **CV = 0.419** (높은 변동성) → 하나의 정책을 일괄 적용하는 것은 비효율적

## 비즈니스 활용

### 1. 최우선 과제: Q2(저CCTV/고범죄) CCTV 확충

- **대상 자치구**: 구로구, 노원구, 은평구, 종로구, 성동구, 용산구
  - CCTV 밀도가 낮으면서 범죄율이 높은 가장 취약한 지역
- Q2와 전체 중앙값의 격차(**−20.9%**)를 해소하는 것만으로 밀도 기준 취약 지역 해소 가능
- **정책 목표**: 2028년까지 구로구·노원구를 전체 중앙값(8.34대) 수준으로 보급

### 2. 사분면별 차별화 전략

CV = 0.419 → 전 자치구 일괄 정책은 비효율적

| 사분면 | 자치구 | 전략 |
|---|---|---|
| **Q1 (고CCTV/고범죄)** | 송파구, 강동구, 영등포구, 중구, 관악구 | 카메라는 충분. 추가 설치보다 **AI 영상 분석·경찰 순찰 연계** 등 운영 고도화 |
| **Q2 (저CCTV/고범죄)** | 구로구, 노원구, 은평구, 종로구, 성동구, 용산구 | **신규 설치 최우선** |
| **Q3 (저CCTV/저범죄)** | 중랑구, 강서구, 양천구, 금천구, 강북구 | 현 수준 유지, 분기별 범죄 추이 모니터링 |
| **Q4 (고CCTV/저범죄)** | 강남구, 서초구, 광진구, 도봉구 | CCTV 운영이 가장 효과적인 **벤치마크 대상** → 강남구의 카메라 배치·모니터링 방식을 구로구·노원구에 적용 |

### 3. 예산 배분 우선순위

| 순위 | 대상 자치구 | 정책 |
|---|---|---|
| 1순위 | 구로구·노원구·은평구·종로구 | CCTV 신규 설치 |
| 2순위 | 송파구·강동구·영등포구 | 운영 고도화 |
| 3순위 | 중랑구·강서구·금천구 | 모니터링 유지 |

> "어디에 먼저, 얼마나"를 판단하는 근거.

## 문제 해결 경험

### 1. 역인과관계 해석의 오류 방지

- **문제**: CCTV와 범죄의 상관계수가 **r = 0.768**로 강한 양의 관계 → 단순 해석하면 *"CCTV가 범죄를 유발한다"* 는 잘못된 결론에 빠질 수 있음
- **해결**: CCTV 설치 상위 5개 자치구 = 범죄 발생 상위 5개 자치구임을 확인하고, **4사분면 분류**로 *"범죄가 먼저 발생한 지역에 CCTV가 뒤따라 설치되는 구조"* 임을 검증
- **결과**: *"CCTV 증가 → 범죄 증가"* 가 아니라 **"범죄 증가 → CCTV 설치"** 라는 역인과 방향을 데이터로 규명

### 2. 자치구 간 편차를 무시한 일괄 분석의 한계

- **문제**: 24개 자치구를 하나의 평균으로 분석하면 지역별 특성이 묻힘 (**CV = 0.419**로 변동성이 높음)
- **해결**:
  - 인구당 CCTV 수 × 인구당 범죄율 기준으로 **4사분면 자동 분류 모델** 구축
  - 자치구별 위험도를 **Z-score로 점수화**
- **결과**: 동일 정책이 아닌 **사분면별 차별화 전략**(Q2는 확충, Q1은 운영 고도화)의 근거 마련

## 실행 방법

### Local 실행

```bash
# 1. 저장소 클론
git clone https://github.com/JungMoonYoung/CCTV-Analysis.git
cd CCTV-Analysis

# 2. 가상환경 및 의존성
python -m venv venv
# Windows
.\venv\Scripts\activate
# macOS/Linux
source venv/bin/activate
pip install -r requirements.txt

# 3. 대시보드 실행
streamlit run dashboard.py
```
