# 기획 배경

> "CCTV는 정말 범죄를 줄일까?"
>
- 서울시는 범죄 예방을 위해 지속적으로 CCTV를 설치하고 있지만, **실제 효과에 대한 데이터 기반 검증이 부족**

- **실제 공공데이터**를 활용한 현실적인 분석으로, 정책 입안자가 활용 가능한 **실무형 인사이트 도출**이 목표

**핵심 발견:**
- **전체 CCTV vs 전체 범죄: r = 0.7684** (p < 0.0001)
  - 강한 양의 상관관계
  - 통계적으로 매우 유의함

- **방범 CCTV vs CCTV효과범죄: r = 0.7408** (p < 0.0001)
  - 절도·강도 범죄와 방범용 CCTV 간 강한 양의 상관관계

**해석:**
- 일반적인 예상(CCTV ↑ → 범죄 ↓)과 달리 **양의 상관관계** 발견
- **역인과관계(Reverse Causality)** 확인: 범죄가 많은 지역에 CCTV를 집중 설치했기 때문

# 분석 방법론

### 1. 탐색적 데이터 분석 (EDA)

- **기술통계량 계산**: 평균, 표준편차, 최소/최대값
- **분포 확인**: 히스토그램, Box Plot으로 이상치 탐지
- **변수 간 관계**: 산점도 행렬(Pair Plot)로 변수 간 관계 시각화

### 2. 상관분석 (Correlation Analysis)

**Pearson 상관계수 계산:**
```python
from scipy import stats

r_corr, p_value = stats.pearsonr(
    merged['CCTV_per_1000'],
    merged['범죄_per_1000']
)
# r = 0.7684, p < 0.0001 (통계적으로 매우 유의)
```

**통계적 가설 검정:**
- **귀무가설 (H0)**: CCTV와 범죄 간 상관관계가 없다 (ρ = 0)
- **대립가설 (H1)**: CCTV와 범죄 간 상관관계가 있다 (ρ ≠ 0)
- **결과**: p-value < 0.0001 → 귀무가설 기각, 유의수준 0.05에서 통계적으로 유의

### 3. 회귀분석 (Regression Analysis) - 예정

**다중 선형 회귀 모델:**
```
범죄율 = β0 + β1*(방범CCTV) + β2*(인구밀도) + ε
```

**회귀 가정 검토:**
- 다중공선성 검사 (VIF)
- 잔차 정규성 검정 (Q-Q Plot)
- 등분산성 검정 (Residual Plot)

### 4. 지역 분류 (Segmentation)

**4사분면 분류 알고리즘:**
```python
cctv_median = df['방범CCTV_per_1000'].median()
crime_median = df['CCTV효과범죄_per_1000'].median()

if cctv >= cctv_median and crime >= crime_median:
    return 'Q1: 고CCTV/고범죄'
elif cctv < cctv_median and crime >= crime_median:
    return 'Q2: 저CCTV/고범죄 (우선순위)'
elif cctv < cctv_median and crime < crime_median:
    return 'Q3: 저CCTV/저범죄'
else:
    return 'Q4: 고CCTV/저범죄 (효과적)'
```

### **2. 인구 규모 통제 문제**

> **문제**
>
> 절대 숫자로 비교 시 인구가 많은 자치구가 과대평가됨
> (강남구 56만명 vs 도봉구 30만명)
>

**해결**

- **인구 천 명당 비율로 정규화**
  - CCTV_per_1000 = (CCTV_총계 / 총인구) * 1000
  - 범죄_per_1000 = (총범죄_발생 / 총인구) * 1000
- **효과**: 인구 규모가 다른 자치구를 공정하게 비교 가능
- **결과**: 중구(인구 적음)가 CCTV 밀도 1위로 드러남

### **3. 상관관계 해석 오류 가능성**

> **문제**
>
> 양의 상관관계 발견 → "CCTV가 범죄를 증가시킨다"로 잘못 해석될 위험
>

**해결**

- **역인과관계(Reverse Causality) 명시**
  - 실제 인과관계: 범죄 많음 → CCTV 추가 설치
  - 관찰된 현상: CCTV 많음 + 범죄 많음 (양의 상관관계)
- **한계점 문서화**
  - 단면 데이터로 인과관계 추론 불가
  - 시계열 데이터 필요 (Before-After 분석)
  - 이중차분법(DID) 사용 권장
- **대시보드에 경고 문구 추가**

  ```python
      return pd.read_csv('data.csv')
  ```
- **효과**: 데이터 로딩 시간 80% 감소 (5초 → 1초)
- **차트 최적화**: Plotly config 설정으로 렌더링 속도 개선

# 프로젝트 한계 및 개선 방향

### 한계점

1. **시계열 데이터 부재**
   - 현재: 2024년 단면 데이터만 사용
   - 한계: 인과관계 추론 불가 (상관관계만 확인)

2. **누락 변수 편향**
   - 고려 못한 변수: 조명, 경찰 순찰, 상권 활성화, 유동인구
   - 영향: 회귀분석 계수의 정확도 저하 가능성

3. **CCTV 효과 측정의 어려움**
   - CCTV 설치 '전후' 비교 불가
   - 범죄 감소가 CCTV 때문인지 다른 요인 때문인지 불명확

### 개선 방향

1. **시계열 분석 추가**
   - 2010년~2024년 연도별 데이터 확보
   - 이중차분법(DID)으로 CCTV 설치 효과 검증

2. **공간 분석 고도화**
   - 개별 사건 좌표 데이터 활용
   - CCTV 반경 200m 내 범죄 발생률 비교

3. **머신러닝 예측 모델**
   - Random Forest로 범죄 발생 예측
   - SHAP 값으로 변수 중요도 해석

4. **추가 변수 수집**
   - 가로등 밀도, 경찰서 위치, 유흥가 분포 등
   - 다중 회귀분석 정확도 향상
