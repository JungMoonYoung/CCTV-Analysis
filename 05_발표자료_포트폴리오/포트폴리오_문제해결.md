## 문제 1: 역인과관계(Reverse Causality) 발견 및 대응

### 📌 상황
초기 가설은 "CCTV가 많을수록 범죄가 줄어들 것"이었으나, 실제 분석 결과 **양의 상관관계(r=0.768)**가 나타남

### 🔍 문제 인식
```
기대: CCTV ↑ → 범죄 ↓ (음의 상관)
현실: CCTV ↑ → 범죄 ↑ (양의 상관, r=0.768, p<0.001)
```

**의문점:**
- CCTV가 범죄를 유발하는가?
- 데이터 오류인가?
- 통계 계산 실수인가?

### 💡 원인 분석 프로세스

#### 1단계: 데이터 검증
```python
# 데이터 품질 확인
print(df.isnull().sum())  # 결측치 없음
print(df['CCTV_총계'].describe())  # 이상치 없음
print(df['총범죄_발생'].describe())  # 정상 범위
```
**결과**: 데이터는 정상, 계산 오류 아님

#### 2단계: 시각화로 패턴 확인
- CCTV 상위 5개 자치구 = 범죄 상위 5개 자치구
- 강남구: CCTV 1위(8,365대) + 범죄 1위(6,107건)

#### 3단계: 인과관계 방향 재검토
**가설 전환:**
```
잘못된 가설: CCTV → 범죄 감소 (인과)
올바른 해석: 범죄 많음 → CCTV 설치 (역인과)
```

### ✅ 해결 방안

**1. 통계적 접근**
- 단면 데이터(Cross-sectional)의 한계 인정
- 인과관계 추론 불가 명시
- 상관관계 ≠ 인과관계 강조

**2. 정책적 해석 제시**
```
실제 메커니즘:
범죄 발생 증가 → 시민 불안 → 정책적 대응 → CCTV 추가 설치
```

**3. 향후 연구 방향 제시**
- 시계열 데이터 확보 필요 (2018-2024)
- 이중차분법(DID) 적용 방안
- CCTV 설치 전후 비교 실험 설계

### 📊 비즈니스 임팩트
- ❌ 잘못된 결론: "CCTV 더 설치하면 범죄 줄어듦" (잘못된 정책)
- ✅ 올바른 결론: "범죄 많은 곳에 CCTV 먼저 설치" (현실 반영)
- 💡 정책 제안: Q2 지역(저CCTV/고범죄)에 우선 투자

### 🎯 배운 점
1. **비판적 사고**: 예상과 다른 결과에 의문 제기
2. **인과 추론 한계**: 데이터 유형에 따른 분석 한계 인식
3. **맥락 이해**: 통계 수치만이 아닌 정책 배경 고려

## 문제 2: 불균형한 자치구 간 격차 처리

### 📌 상황
자치구별 인구가 5배 차이 (종로구 13만명 vs 송파구 65만명), CCTV 대수로만 비교 시 왜곡 발생

### 🔍 문제 인식
```
강남구: 8,365대 (1위) vs 도봉구: 2,820대 (24위)
→ 강남구가 CCTV를 3배 더 설치했는가?
```

**실제 인구:**
- 강남구: 562,382명
- 도봉구: 303,770명

**문제점**: 절대 수치 비교는 인구 규모 차이를 무시함

### 💡 해결 과정

#### 1단계: 정규화(Normalization) 필요성 인식
```python
# 문제가 있는 비교
df_sorted = df.sort_values('CCTV_총계', ascending=False)

# 올바른 비교
df['CCTV_per_1000'] = (df['CCTV_총계'] / df['총인구']) * 1000
df['범죄_per_1000'] = (df['총범죄_발생'] / df['총인구']) * 1000
```

#### 2단계: 정규화 전후 비교
| 자치구 | CCTV 대수 | 순위 (절대) | CCTV/천명 | 순위 (정규화) |
|--------|----------|------------|-----------|--------------|
| 중구 | 3,276대 | 17위 | **25.4대** | **1위** |
| 강남구 | 8,365대 | **1위** | 14.9대 | 6위 |
| 도봉구 | 2,820대 | **24위** | 9.3대 | 15위 |

**인사이트**: 중구가 실제로 CCTV 밀도 1위!

#### 3단계: 모든 분석에 정규화 적용
```python
# 상관분석
corr = df['CCTV_per_1000'].corr(df['범죄_per_1000'])

# 4사분면 분류
cctv_median = df['방범CCTV_per_1000'].median()
crime_median = df['CCTV효과범죄_per_1000'].median()
```

### ✅ 해결 방안
1. **인구 천명당 정규화**: 공정한 비교 가능
2. **표준화(Z-score)**: 위험도 점수 산정
3. **상대 비교**: 중앙값 기준 분류

### 📊 비즈니스 임팩트
- ✅ 공정한 자치구 간 비교
- ✅ 실제 투자 우선순위 정확히 파악
- ✅ Q2 지역 4개 구 도출 (동대문·마포·양천·강서)

### 🎯 배운 점
1. **정규화의 중요성**: 비교 가능한 척도 만들기
2. **데이터 탐색**: 절대 수치의 함정 인식
3. **공정성**: 인구 규모 차이 고려한 분석

## 문제 3: 획일적 정책의 비효율성 발견

### 📌 상황
"모든 자치구에 CCTV를 동일하게 20% 증설하자"는 제안 검토 요청

### 🔍 문제 인식
변동성 분석 결과:
```python
cv_crime = df['범죄_per_1000'].std() / df['범죄_per_1000'].mean()
# CV = 0.419 (높은 변동성)
```

**의미**: 범죄율 표준편차가 평균의 41.9%
→ 자치구 간 범죄 발생 패턴이 크게 다름

### 💡 원인 분석

#### 1단계: 변동계수(CV) 비교
| 변수 | CV | 해석 |
|------|-----|------|
| 범죄율 | **0.419** | 변동성 가장 큼 |
| 인구 | 0.305 | 중간 |
| CCTV 밀도 | 0.297 | 중간 |

**인사이트**: 범죄는 지역 특성에 따라 크게 달라짐

#### 2단계: 4사분면 분류로 이질성 확인
| 분면 | 자치구 수 | 평균 CCTV | 평균 범죄 | 필요 정책 |
|------|----------|----------|----------|-----------|
| Q1 | 8개 | 10.89대 | 4.94건 | 종합 대책 |
| Q2 | 4개 | 6.60대 | 4.02건 | CCTV 추가 |
| Q3 | 8개 | 6.50대 | 2.95건 | 현상 유지 |
| Q4 | 4개 | 10.21대 | 3.46건 | 벤치마킹 |

**인사이트**: 각 분면마다 전혀 다른 정책 필요

#### 3단계: 비효율성 정량화
**시나리오: 모든 구에 20% 증설**
- Q3 (저범죄): 불필요한 예산 낭비
- Q1 (고CCTV/고범죄): 이미 많은데 효과 없음
- Q2 (저CCTV/고범죄): 부족분 해결 안 됨

**개선안: 지역 맞춤형 투자**
- Q2에 집중 투자: 40억원
- 예상 효과: 320건/년 범죄 감소

### ✅ 해결 방안
**데이터 기반 의사결정 프레임워크:**
```
1. 변동성 분석 (CV) → 획일적 정책 부적합 판단
2. 4사분면 분류 → 지역별 우선순위 도출
3. 투자 효율 계산 → Q2 최우선 선정
4. ROI 산정 → 예산 정당화
```

### 📊 비즈니스 임팩트
- ❌ 획일적 정책: 예산 낭비, 효과 미미
- ✅ 맞춤형 정책: 40억원 집중 투자 → 320건 범죄 감소
- 💰 ROI: 3년 내 투자 회수 (범죄 1건당 사회적 비용 500만원 기준)

### 🎯 배운 점
1. **데이터 다양성**: 평균만이 아닌 분산 분석
2. **세그멘테이션**: 집단 내 이질성 파악
3. **효율성**: 자원 배분의 우선순위 설정

## 문제 4: 설명력 한계 극복 (R² = 0.590)

### 📌 상황
선형회귀 결과 R² = 0.590
→ CCTV가 범죄의 59%만 설명, 나머지 41%는?

### 🔍 문제 인식
```python
from scipy.stats import linregress
slope, intercept, r_value, p_value, std_err = linregress(
    df['CCTV_per_1000'], df['범죄_per_1000']
)
r_squared = r_value ** 2  # 0.590
```

**질문**: 나머지 41%를 설명하는 변수는?

### 💡 원인 분석

#### 1단계: 누락 변수(Omitted Variables) 탐색
**가능한 요인:**
- 가로등 조명 밝기
- 경찰 순찰 빈도
- 유흥가 밀집도
- 유동인구
- 빈곤율

**문제**: 데이터 확보 불가

#### 2단계: 설명 가능한 변수 추가 시도
```python
# 인구밀도 추가
X = df[['CCTV_per_1000', '인구밀도']]
y = df['범죄_per_1000']

# 다중회귀
model = sm.OLS(y, sm.add_constant(X)).fit()
# R² 증가 미미
```

#### 3단계: 한계 인정 및 투명한 커뮤니케이션
**보고서에 명시:**
```
CCTV가 범죄의 59%를 설명
나머지 41%는 조명, 순찰, 유흥가 등 다른 요인
→ CCTV만으로는 충분하지 않음
```

### ✅ 해결 방안
**1. 정직한 한계 명시**
- R²=0.590 → CCTV가 주요 변수지만 유일 변수는 아님
- 단면 데이터의 한계 인정

**2. 정책 제안 조정**
- Q1 지역: CCTV 외 **종합 방범 대책** 필요
- 조명 개선, 순찰 강화 병행

**3. 향후 연구 방향**
- 다른 변수 데이터 확보
- 다층 분석(Multilevel Model)
- 공간 분석(Spatial Analysis)

### 📊 비즈니스 임팩트
- ✅ 현실적 정책 제안: CCTV + 종합 대책
- ✅ 이해관계자 신뢰: 한계 투명하게 공개
- ✅ 과대 평가 방지: "CCTV가 만능"이라는 오해 차단

### 🎯 배운 점
1. **완벽주의 극복**: 59% 설명력도 충분히 의미 있음
2. **투명성**: 한계를 숨기지 않고 명시
3. **현실성**: 데이터 분석의 한계 인정

## 문제 5: 중앙값 vs 평균값 선택 딜레마

### 📌 상황
4사분면 분류 기준선을 정할 때: 평균(mean) vs 중앙값(median)?

### 🔍 문제 인식
```python
# 방범CCTV 분포 확인
print(df['방범CCTV_per_1000'].mean())    # 8.56대/천명
print(df['방범CCTV_per_1000'].median())  # 8.34대/천명
print(df['방범CCTV_per_1000'].describe())

# 이상치 확인
중구: 25.4대/천명 (극단치)
```

**질문**: 어떤 기준선이 더 적절한가?

### 💡 원인 분석

#### 1단계: 이상치 영향 평가
```python
# 중구 제거 전후 비교
mean_with = df['방범CCTV_per_1000'].mean()     # 8.56
mean_without = df[df['자치구']!='중구']['방범CCTV_per_1000'].mean()  # 8.12

# 평균이 이상치에 민감함
```

#### 2단계: 중앙값 장점
- ✅ 이상치에 강건(Robust)
- ✅ "절반보다 많다/적다" 직관적 해석
- ✅ 정책 목표 설정 용이: "모든 구를 중앙값 이상으로"

#### 3단계: 평균값 단점
- ❌ 중구(25.4) 때문에 기준선이 높아짐
- ❌ Q2 지역이 줄어듦 → 우선순위 왜곡

### ✅ 해결 방안
**중앙값(Median) 채택 근거:**
1. **통계적**: 이상치 존재 시 중앙값이 더 robust
2. **정책적**: "상위 50%가 되자" 목표 명확
3. **실무적**: 8.34대/천명이 현실적 기준

**결과:**
```python
cctv_median = df['방범CCTV_per_1000'].median()  # 8.34
crime_median = df['CCTV효과범죄_per_1000'].median()  # 3.74

# 4사분면 분류
Q2 지역: 4개 구 (동대문·마포·양천·강서)
```

### 📊 비즈니스 임팩트
- ✅ 정확한 우선순위: Q2 4개 구 도출
- ✅ 현실적 목표: 중앙값 수준 달성
- ✅ 정책 정당화: "절반보다 낮은 구에 투자"

### 🎯 배운 점
1. **통계 선택의 중요성**: 상황에 맞는 척도 선택
2. **이상치 처리**: 제거보다 robust 통계량 사용
3. **의사소통**: 중앙값의 직관적 해석
