# SRS v1.0 — 서울시 CCTV 설치 현황과 범죄 발생 상관 분석 프로젝트

## 1. 개요 (Introduction)

### 1.1 목적 (Purpose)

본 프로젝트의 목적은 서울시 행정구 단위의 CCTV 설치 현황과 범죄 발생 건수 간의 관계를 분석하고,
그 결과를 바탕으로 도시 안전 정책 방향 및 CCTV 설치 전략에 대한 인사이트를 도출하는 것이다.

단순히 "CCTV 많으면 범죄가 줄어든다" 수준을 넘어서,
인구수·유동인구 등을 보정한 상대적 지표를 사용하여,
CCTV 설치가 과소/과다한 지역을 식별하고, 정책 제안까지 연결하는 것을 목표로 한다.

최종 산출물은 분석 보고서(PDF 또는 Markdown 기반 문서) 형식이며,
필요시 GitHub 리포지토리로 공개하여 포트폴리오로 활용한다.

**주요 타겟 독자**: 데이터 분석가 채용 담당자/면접관 - 기술적 깊이와 문제해결 능력 중심

### 1.2 범위 (Scope)

본 프로젝트는 다음 작업 범위를 포함한다.

#### 포함(✔)

- 서울시 내 행정구(또는 경찰서 관할구역) 단위 분석
- CCTV 설치 현황 데이터 수집 및 정리
  - **CCTV 유형별 분류**: 방범용, 교통단속용, 어린이안전용, 기타
- **분석 대상 범죄**: CCTV 억제 효과가 예상되는 범죄만 선별
  - 절도 (Theft)
  - 강도 (Robbery)
  - 차량 관련 범죄
  - 공공장소 폭력 (주취·거리폭력) - 부분 포함
  - 성범죄 - 별도 분리 분석
- **분석 기간**: 2023년 최신 데이터 기준 단면 분석
  - 시계열 데이터는 외부 자료(서울시 보고서, 뉴스) 인용으로 보완
- 보정 변수(인구수, 유동인구) 활용한 표준화된 범죄율 및 CCTV 밀도 계산
- CCTV 밀도와 범죄율의 상관분석, 다중 회귀모형
- 공간 시각화(구 단위 Choropleth 지도 등)
- 정책/운영 제안이 포함된 분석 보고서 작성

#### 제외(❌)

- 개별 사건 단위의 정확한 좌표 기반 분석 (개인정보 문제)
- 실시간 데이터 수집 및 대시보드 구축
- 심화 머신러닝(복잡한 예측 모델, 딥러닝 기반 이상탐지 등)
- 연도별 시계열 회귀분석 (데이터 제약)

### 1.3 이해관계자 (Stakeholders)

- **분석자(본인)**: 데이터 수집, 모델링, 해석, 보고서 작성의 전 과정을 수행
- **잠재 독자**
  - 데이터 분석가/데이터사이언티스트 면접관 (1차 타겟)
  - 도시/치안 정책 담당자를 가정한 가상의 이해관계자
  - 포트폴리오 검토자(채용 담당자)

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 데이터 수집 (FR-1)

#### FR-1.1 CCTV 데이터 수집
- 서울 열린데이터광장에서 "CCTV 설치 현황" 관련 데이터셋 수집
- **최소 포함 필드**:
  - 행정구 또는 동 이름
  - CCTV 대수
  - CCTV 설치 목적(방범, 교통단속, 어린이안전 등)
  - (선택) 좌표 정보

#### FR-1.2 범죄 통계 데이터 수집
- 공공데이터포털 또는 경찰청 데이터포털에서 제공하는 지역별 범죄 통계 수집
- 연도: 2023년 기준
- 자치구 또는 경찰서 관할구 단위
- **최소 필드**:
  - 절도, 강도, 차량범죄, 공공장소 폭력, 성범죄 건수
  - 총 범죄 건수 (참고용)

#### FR-1.3 보정 변수 수집 (선택)
- 서울시 인구/인구밀도 데이터 (자치구 단위, 2023년)
- (선택) 유동인구/상권 관련 데이터

#### FR-1.4 GIS 경계 데이터 수집
- 서울시 행정구/자치구 경계 GeoJSON 또는 SHP 파일 수집
- 공간 시각화를 위한 기본 레이어로 사용

### 3.2 데이터 전처리 및 통합 (FR-2)

#### FR-2.1 행정구 단위 통합
- CCTV 데이터와 범죄 데이터를 공통 키(예: 자치구명) 기반으로 병합
- 구 이름 표기가 다를 경우 매핑 테이블을 사용해 통일

#### FR-2.2 지표 생성
아래와 같은 파생 변수 생성:
- `CCTV_대수_합계`: 자치구별 CCTV 총 수
- `방범용_CCTV`: 방범 목적 CCTV만 집계
- `인구당_CCTV`: CCTV 대수 / 인구수 × 1000
- `면적당_CCTV` (선택): CCTV 대수 / 면적(㎢)
- `인구당_범죄율`: (CCTV 효과 범죄 건수 합계) / 인구수 × 1000
- `유형별_범죄율`: 절도율, 강도율, 차량범죄율 등

#### FR-2.3 결측치 처리
- 인구 또는 CCTV 데이터가 유실된 구가 있다면:
  - 제외하거나,
  - 평균/이웃값 사용 여부를 기록 후 처리

### 3.3 분석 기능 (FR-3)

#### FR-3.1 기초 통계 및 상관분석
- CCTV 대수, 인구당 CCTV, 인구당 범죄율 간의 상관계수(Pearson, Spearman) 계산
- CCTV 유형별(방범/교통/어린이) 상관분석 별도 수행
- 상위/하위 5개 구 리스트를 통해 "극단값 구" 식별

#### FR-3.2 시각화
- **Bar Chart**: 자치구별 CCTV 대수, 범죄 건수 비교 (유형별 누적)
- **Scatter Plot**: 인구당 CCTV vs 인구당 범죄율
- **Heatmap**: 상관 행렬 시각화
- **Choropleth 지도**:
  - 자치구별 CCTV 밀도 지도 (방범용 vs 전체)
  - 자치구별 범죄율 지도 (CCTV 효과 범죄만)
  - 두 지도를 비교

#### FR-3.3 회귀분석
- **다중 선형 회귀**:
  - 종속 변수: 인구당 범죄율 (CCTV 효과 범죄)
  - 독립 변수: 인구당 방범CCTV, 인구밀도, 유동인구(선택)
- **회귀 가정 검토**:
  - VIF(다중공선성 검사)
  - 잔차 정규성 Q-Q plot
  - 잔차 vs 예측값 산점도
- **회귀 결과**:
  - 회귀계수, p-value, Adjusted R² 제시
  - "방범CCTV가 많을수록 범죄율이 낮다는 통계적 근거 여부" 평가
  - 한계점: 역인과성(범죄 많은 곳에 CCTV 추가 설치) 가능성 논의

### 3.4 정책 인사이트 도출 (FR-4)

#### FR-4.1 지역 분류
- **CCTV 대비 범죄율이 높은 지역**: "고위험/저커버리지 지역" → CCTV 추가 설치 우선
- **CCTV는 많은데 범죄도 많은 지역**: "고위험/고커버리지 지역" → CCTV 외 대안 필요
- **CCTV 대비 범죄율 낮은 지역**: "저위험/고커버리지 지역" → 현상 유지
- **CCTV 적고 범죄도 낮은 지역**: "저위험/저커버리지 지역" → 불필요 확충 지양

이를 기반으로 4개 사분면 분류표 또는 클러스터링.

#### FR-4.2 정책 제안
- CCTV 추가 설치 우선지역 제안 (고위험/저커버리지)
- CCTV 유형별 효과 분석:
  - 방범용 CCTV가 범죄 억제에 유의미한지
  - 교통단속용은 범죄율과 무관할 가능성
- 단순 CCTV 확충 외에 고려해야 할 대안:
  - 가로등/조명 개선
  - 경찰 순찰 강화
  - 생활인프라(상권, 공원) 개선
- **한계점 명시**:
  - "CCTV만으로 범죄 예방을 단정할 수 없음"
  - 역인과성 문제 (범죄 많은 곳에 CCTV 더 설치)
  - 단면 데이터 한계 (인과 추론 불가)

### 3.5 보고서 작성 (FR-5)

최종 결과를 보고서 형태(Markdown → PDF)로 정리

**구조 예시**:
1. 연구 배경 및 목적
2. 데이터 설명 및 수집 과정
3. 데이터 한계 및 제약 사항
4. 분석 방법 (전처리, 지표 정의, 회귀모형)
5. 결과
   - 기초 통계
   - 시각화 (그래프 + 지도)
   - 회귀분석 결과
6. 정책 인사이트 및 제안
7. 한계점 및 향후 연구 제안

- 시계열 변화는 서울시 행정 보고서와 뉴스 기반 보조 근거로 제시

## 6. 평가 기준 (Evaluation)

분석 자체에 대한 정량적인 성능 지표보다는,
**"해석의 논리성과 시각화의 설득력"**이 핵심.

### 정량적 기준 예시:
- **설명력**: 다중 회귀 모델의 Adjusted R² ≥ 0.3 이상이면 참고 가능 수준
- **상관계수**: 인구당 방범CCTV vs 인구당 범죄율의 상관계수가 유의미한지 (p-value < 0.05)
- **다중공선성**: VIF < 10 (독립변수 간 과도한 상관 없음)

### 보고서 완성도:
- 최소 **15페이지** 이상의 구조화된 분석 보고서
- 주요 시각화(지도 포함) **7개 이상**
- 정책 제안 섹션 포함
- 한계점 및 향후 연구 방향 명시

## 7. 리스크 및 대응 (Risk Management)

| 리스크 | 내용 | 대응 방안 |
|--------|------|-----------|
| CCTV 데이터에 좌표/세부정보 미비 | 위치 정보가 없다면 지도 분석이 제한됨 | 자치구 단위 집계로 전환, 또는 지오코딩 사용 |
| **CCTV 유형 구분 데이터 미비** | 방범/교통/주차 구분이 없을 수 있음 | 전체 CCTV로 통합 분석, 한계점 명시 |
| 범죄 데이터가 전체 범죄만 제공 | CCTV 효과 범죄 유형별 분리 불가 | 전체 범죄 건수 기준으로 분석, 한계 명시 |
| 인구 데이터 연도 불일치 | CCTV/범죄 데이터 연도와 인구 데이터 연도가 다를 수 있음 | 가능한 한 가장 근접한 연도 사용, 한계점으로 명시 |
| **시계열 데이터 불일치** | 연도별 CCTV 증설 이력 부재 | 단면 분석 중심, 증가 추세는 문헌 인용 |
| 시간 부족(14일 제한) | 모든 아이디어를 다 구현하기 어려움 | 필수 분석(상관 + 지도 + 회귀)만 먼저 완료, 추가 분석은 향후 개선 과제로 분리 |

## 8. 성공 기준 (Success Criteria)

✅ CCTV 및 범죄 데이터를 이용해 자치구 단위 분석을 완료하였다.

✅ 최소 **2개 이상의 지도 시각화**와 **5개 이상 그래프**를 포함한 분석 보고서를 작성하였다.

✅ **CCTV 유형별 효과 차이**를 분석하였다 (방범용 vs 전체).

✅ **회귀분석의 가정 검토**(VIF, 잔차 검정)를 수행하고 결과를 해석하였다.

✅ CCTV 밀도와 범죄율 간의 관계에 대해,
"강한 인과 관계"가 아니더라도 **데이터 기반의 해석과 한계**를 명확히 설명하였다.

✅ **4개 사분면 기반 지역 분류** 및 정책 제안을 포함하였다.

✅ GitHub 또는 문서 형태로 **제3자가 이해할 수 있는 형태의 결과물**이 정리되었다.
